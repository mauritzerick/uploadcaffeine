## What Are Micro-Interactions?

Micro-interactions are the small, functional animations and feedback mechanisms that make interfaces feel alive:

- Button hover states
- Loading animations
- Success/error messages
- Sound effects
- Cursor interactions

## Implementation Tips

### 1. Use Framer Motion

```tsx
<motion.button
  whileHover={{ scale: 1.05 }}
  whileTap={{ scale: 0.95 }}
  transition={{ type: "spring", stiffness: 400 }}
>
  Click Me
</motion.button>
```

Framer Motion makes complex animations simple.

### 2. Respect Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

Always respect user preferences for accessibility.

### 3. Sound Design

Subtle UI sounds enhance feedback:
- Click: high-pitched beep
- Success: ascending chord
- Error: descending tone

Keep them short (`<300ms`) and optional.

### 4. Loading States

Never show a blank screen:
- Skeleton loaders
- Progress indicators
- Animated placeholders

## Common Mistakes

### 1. Too Much Movement
Every element animated = chaotic and distracting.

**Solution**: Animate 1-2 key elements per screen.

### 2. Slow Animations
Animations over `400ms` feel sluggish.

**Solution**: Keep most under `300ms`.

### 3. No Purpose
Animation for animation's sake is annoying.

**Solution**: Every animation should serve a purpose.

### 4. Broken on Mobile
Complex animations can lag on mobile devices.

**Solution**: Test on actual devices, simplify for mobile.

## Measuring Impact

How do you know micro-interactions are working?

### Qualitative

- User feedback ("This is fun to use!")
- Social shares (screenshots of effects)
- Return visits

### Quantitative

- Time on page (longer = more engaged)
- Conversion rate (do they donate?)
- Bounce rate (do they leave immediately?)

## The Balance

Great micro-interactions are:
- **Purposeful** - Serve a clear function
- **Subtle** - Don't overwhelm
- **Consistent** - Follow a pattern
- **Performant** - Don't cause lag
- **Accessible** - Work for everyone
